# SQL Application Project

## Overview
This project is a structured SQL-based application designed to manage various components such as user interfaces, navigation, themes, and shared resources. It includes scripts for setting up the application environment, creating and deleting applications, and managing shared components.

## Project Structure
The project is organized as follows:

- **install.sql**: Main installation script that references other SQL scripts for setting up the application.
- **application/**: Contains SQL scripts for managing the application, including environment setup, user interfaces, and shared components.
  - **deployment/**: Scripts related to deployment options and checks.
  - **pages/**: SQL scripts for individual pages of the application.
  - **shared_components/**: Scripts for shared resources such as files, navigation, security, and themes.
- **readable/**: Contains YAML files and other readable formats for documentation or configuration.

## Installation
To install the application, execute the `install.sql` script in your SQL environment. This script will:

1. Set up the environment.
2. Create the application.
3. Configure user interfaces and shared components.

### Steps:
1. Open your SQL client.
2. Load the `install.sql` script.
3. Execute the script to complete the installation.

## Usage
Once installed, the application will be ready to use. The scripts in the `application/` folder manage various aspects of the application, such as navigation, themes, and user interfaces.

## Notes
- Ensure that your SQL environment is properly configured before running the scripts.
- Review the `install.sql` file to understand the sequence of operations.
